# Battle Atlas

The Battle of Gettysburg (July 3, 1863 — Pickett's Charge) as a
desktop-first interactive reconstruction for Apple Silicon macOS, in two
connected experiences:

- **Atlas Mode** — a real-time, scrubbable documentary map of the July 3
  battlefield: 190 units on real LiDAR terrain, unit identity/activity/
  provenance on tap, moment markers, and an honest documented-versus-
  reconstructed distinction throughout.
- **Soldier View** — a fixed first-person viewpoint inside Pickett's
  Charge (*With Garnett's Line*, the Emmitsburg Road to the Angle,
  15:16–15:27), pre-rendered offline in HDRP as seekable video and
  slaved to the same battle clock. Enter, play, pause, seek, exit — the
  camera never steers. Contains a first-entry content warning: explicit,
  soberly presented battlefield violence
  (`docs/reconstruction/violence-and-representation.md`).

**Status:** Angle Reconstruction V2 phases 0–11 complete (all build
gates closed); Phase 12 (review and release) in progress. Plan of
record: `docs/superpowers/plans/2026-07-08-angle-reconstruction-v2.md`.
Pre-V2 state is preserved at tag `v2-baseline`; the earlier iPhone
sand-table direction lives in `docs/superpowers/specs/`.

## Layout

- `app/` — Unity 6 (HDRP) macOS project: Atlas scene, Soldier View
  player, offline render pipeline
- `pipeline/` — Python: USGS 1 m LiDAR → heightmaps (macro +
  true-scale Angle crop) and the sourced environment bake
- `reconstruction/` — Python: V2 schemas, validators, claims compiler,
  audio/captions generators, asset-license tooling
- `tool/` — browser app for authoring battle data over georeferenced maps
- `data/` — gitignored; regenerated by the pipeline (below)
- `docs/` — plans, gate evidence, runbooks, editorial decisions (see the
  docs index below)

## Fresh-clone setup

The checkout is three layers: **source** (this repository), **required
generated terrain** (built locally from cached USGS data), and
**optional pre-rendered media** (a GitHub Release download).

Prereqs: [uv](https://docs.astral.sh/uv/), Unity 6000.4.11f1 with macOS
Build Support.

### 1. Source checkout (this repo)

Everything historical, deterministic, and testable is committed: the
battle data, the compiled Angle bundle, viewpoint metadata, captions,
third-party assets with licenses, and all code.

### 2. Required: generated terrain (~1.9 GB USGS download, cached)

```bash
cd pipeline
uv sync
uv run python -m terrain_pipeline.cli fetch        # USGS 1 m DEM cache (once)
uv run python -m terrain_pipeline.cli build        # macro heightmap
uv run python -m terrain_pipeline.cli crop         # true-scale Angle crop
uv run python -m terrain_pipeline.cli environment  # sourced environment bake
```

Then open `app/` in Unity and run **BattleAtlas ▸ Import Heightmap**
once — this rebuilds the gitignored
`Assets/Generated/GettysburgTerrain.asset` the Atlas scene references —
and save the scene. Open `Assets/Scenes/Atlas.unity` and press Play for
Atlas Mode.

### 3. Optional: Soldier View media (~2 GB download)

Pre-rendered media is a release artifact, never a Git object (plan
§10.1). Download `garnett-road-to-angle.full.mp4` (1440p30, 1.67 GB) and
`garnett-road-to-angle.proxy.mp4` (720p30, 331 MB) from the
**`soldier-view-media-v1` GitHub Release** (forthcoming — the owner
publishes it at the close of Phase 12; until then the files live in the
maintainer's `docs/benchmarks/captures/p10-gate/`), verify them against
`docs/benchmarks/captures/p10-gate/p10-release-manifest.json` (sha256),
and place both in `app/Assets/StreamingAssets/SoldierView/`.

Without the media the app still runs and degrades honestly: entry is
refused with a clear message, or falls back to the proxy (with an
on-screen badge) if only that file is present. Media can also be
re-rendered from source on a 16 GB+ Apple Silicon machine (~2 h):
`docs/reconstruction/render-runbook.md`.

## Authoring tool

`tool/` is a browser app for authoring battle data: trace unit paths
over a georeferenced map (with scanned historical maps as overlays),
edit keyframes with provenance, preview interpolated movement, and
export schema-validated battle JSON (see `docs/format/battle-format.md`).

```bash
cd tool
npm install
npm run dev    # http://localhost:5180 — load data/heightmap/heightmap.json to begin
npm test
```

The tool autosaves to browser localStorage as a crash net; exported JSON
files are the durable, validated artifact — export early and often.

## Tests

- Pipeline: `cd pipeline && uv run pytest -q`
- Reconstruction (V2): `cd reconstruction && uv run pytest -q`
- Authoring tool: `cd tool && npm test`
- Unity EditMode (editor closed): `"$UNITY" -batchmode -projectPath "$(pwd)/app" -buildTarget OSXUniversal -runTests -testPlatform EditMode -testResults "$(pwd)/app/test-results.xml" -logFile -`
- Unity PlayMode (editor closed; Soldier View sync): same command with
  `-testPlatform PlayMode`. Media-dependent tests need the dev proxy
  (`reconstruction/scripts/generate_dev_proxy.sh`) and, for the
  production-media seek tests, the release media staged as above.
