# Battle Atlas

An iPhone demo: the Battle of Gettysburg as an interactive 3D "sand table" —
real LiDAR terrain, a scrubbable timeline of troop movements, and hand-crafted
first-person moments. See `docs/superpowers/specs/` for the full design.

**Status:** Phase 1 (terrain proof) complete — real Gettysburg terrain flying
on-device with map-style touch controls. Tag: `phase1-terrain-proof`.

## Layout

- `pipeline/` — Python: USGS 1 m LiDAR → Unity heightmap (`data/heightmap/`)
- `app/` — Unity 6 (URP) iOS app
- `data/` — gitignored; regenerated by the pipeline (below)
- `docs/superpowers/` — design spec and implementation plans

## Fresh-clone setup

Prereqs: [uv](https://docs.astral.sh/uv/), Unity 6 LTS with iOS Build Support, Xcode.

1. Build the terrain data (~1.9 GB download from USGS, cached in `data/dem_cache/`):

   ```bash
   cd pipeline
   uv sync
   uv run python -m terrain_pipeline.cli fetch
   uv run python -m terrain_pipeline.cli build
   ```

2. Open `app/` in Unity. The committed scene references a gitignored generated
   terrain asset, so on first open run **BattleAtlas ▸ Import Heightmap**
   (menu bar) — this rebuilds `Assets/Generated/GettysburgTerrain.asset` and
   replaces the scene's terrain object. Save the scene.

3. iOS: File ▸ Build Profiles ▸ iOS ▸ Build, then open the generated
   `Unity-iPhone.xcodeproj`, set your signing team, run on device.

## Authoring tool

`tool/` is a browser app for authoring battle data: trace unit paths over a
georeferenced map (with scanned historical maps as overlays), edit keyframes
with provenance, preview interpolated movement, and export schema-validated
battle JSON (see `docs/format/battle-format.md`).

```bash
cd tool
npm install
npm run dev    # http://localhost:5180 — load data/heightmap/heightmap.json to begin
npm test
```

## Tests

- Pipeline: `cd pipeline && uv run pytest -q`
- Authoring tool: `cd tool && npm test`
- Unity (editor closed): `"$UNITY" -batchmode -projectPath "$(pwd)/app" -runTests -testPlatform EditMode -testResults "$(pwd)/app/test-results.xml" -logFile -`

Note: terrain renders with 2.5× vertical exaggeration (sand-table convention),
set in `app/Assets/Editor/HeightmapImporter.cs`. Raw elevation data is unexaggerated.
