## Design decisions locked here

1. **No format changes. Zero.** Everything below authors against the existing contract (`battle-format.md`: units/keyframes/parent/regiments/events/environment). A format task exists (Task 1) only to add two *documentation* paragraphs — no schema, tool, or loader change, no version bump. Rationale per decision:
   - **HQ / hospital / provost markers: DEFERRED to a later phase.** The unit model is a combat-formation model (side, frontage, formation, strength, a flag flown per unit every frame); rendering a hospital as a side-colored block with a battle flag *misrepresents* — the exact sin this phase exists to fix. Doing them honestly needs a `kind` field plus renderer work (non-combat marker visuals), which is format + Unity surface for 8–9 of ~127 units. The slice value is in the ~118 fighting formations; the survey §3.4 already holds the deferred phase's complete research base (Meade's Leister→Powers Hill track, Lee's HQ, Powers Hill node, the hospitals). **Meade's HQ move defers with them.** The one §3.4-adjacent body that IS authored now: the **Artillery Reserve park + ammunition train** — it is an organized artillery command (Tyler; 2,375 men) with an A− documented mid-window displacement, and it renders honestly as a unit.
   - **Off-map engagements (East Cavalry Field): research-doc-only; NO `offMapNotes[]` format element.** Nothing in the app consumes annotations yet — adding format surface for an unrendered field is speculative contract. The survey's ruling ("annotate, never fake") is satisfied by §3.1/§3.4 of the research doc plus a "Deferred" section in this plan; when the UI phase builds an annotation layer, it defines the format element it actually needs. No unit is placed for Gregg/Custer/Stuart, Fairfield, Westminster, or Cashtown — silent omission is prevented by the research doc and the Done section, not by JSON.
   - **Documented silence: absence of events IS the encoding; the silence citation lives in the unit's keyframe citations.** No new field. A tablet like Jones's "Not actively engaged" or the XII Corps' fire "until 10:30 A.M." is simultaneously the position attestation and the negative attestation — one string on the `t:0` keyframe carries both, and "says who that they did NOT fire" is answerable from the unit itself. Putting silence anywhere else (a `silent: true` flag, a zero-length event) would invent machinery for what the event model already expresses by absence. Task 1 documents this convention in `battle-format.md` so future authors don't "fix" quiet units by inventing fire.
   - **CSA artillery battalions replace the smoke segment ATOMICALLY.** The format doc's migration clause ("when batteries are authored as units, their events migrate to the `unitId` form") executes in Task 5: the 15 battalion units + their cited `artillery_fire` events land **in the same commit that deletes `csa-seminary-bombardment`** — the bombardment never has zero emitters at any commit; smoke never regresses. Documented internal splits (Carter's rifles detached to the railroad cut; Raine's 20-pdr section) keep the battalion as ONE unit at its tablet position and attach the detachment's fire as a **fixed-segment event at the detachment's attested position** — the segment emitter form survives for exactly this.
   - **Attach level & family rules: already locked** (format doc "Engagement events" + "Parent / children"). No decomposition below the survey's attested grain: brigades stay brigades, no invented regiments outside the assault corridor. Where the survey's own grain is finer (sheet 8 draws Kilpatrick at regiment level; Bachman/Reilly are the attested SCF movers inside Henry's battalion), Task 8 may use `parent` children — attested grain rules, in both directions. Never the same fire at two family levels (per-wave review check).
   - **Static units: two keyframes** (`t:0` and `t:endTime=10800`), same position/facing/strength, confidence mapped from the survey grade (A/A− → `documented` with citation; B+/B → position `documented` per tablet, with the silence/negative in the citation text; C → `inferred`, and the survey's two do-not-invent C items — CSA ordnance-train parks, Ewell's hospital farms — are NOT authored). **Movers get their attested keyframes:** Cooper's B/1st PA 3 PM (t=7200), Robinson's two brigades 3 PM (t=7200), Shaler ~3:30 (t=9000), Wright's advance-and-recall ~15:30–16:00, the Artillery Reserve displacement mid-cannonade, Wheeler's ~4 PM move tight against the window end. House conventions hold: Bachelder positions are `documented` positions with `inferred` times; disagreements ride in citation text, never reconciled (Merritt's 11:00/13:00/15:00 arrival spread; sheet-8 "Daniels"/"Rank" vs the OOB).

**Id conventions:** batteries `us-btty-phillips`; CSA battalions `csa-bn-alexander` (the `bn-` prefix also dodges the existing `csa-garnett` infantry brigade); brigades follow the existing bare pattern (`us-candy`, `csa-doles`); acting commanders get the survey's names (`csa-perrin`, not McGowan); SCF `us-cav-farnsworth`, `us-btty-elder`. **Frontage heuristic** for units the record doesn't measure: infantry ≈ strength × 0.27 m (consistent with the existing authored brigades), batteries 80–120 m, battalion gun lines 250–500 m read off the sheet — a presentation convention, noted once in the generator lib, not an attestation claim.

- **Authoring throughput, not code, is the long pole.** ~127 units × citation-grade table rows is the cost center; the generator keeps it a table-editing problem and each wave ships alone — if the phase pauses mid-way, waves 1–3 already transformed the bombardment.
- **Coordinate quality off sheet 8:** positions are position-*classes*; tracing error inside a sector is real but bounded (the sector is the claim). Convention already carries it (documented position, inferred precision noted); the provenance reviews spot-check against the §1 landmark table.
- **Perf at ~192 markers** is judged cheap but unverified until Task 2 — which is why Task 2 precedes all authoring. Fallback (instanced block markers) is a known pattern and stays out of scope unless the fixture check fails.
- **Smoke-budget clamp at the new cannonade peak:** ~600 live puffs fits the buckets on paper; the overflow warning is loud by design — if it fires, tune cadence/life in the params structs, not buffer sizes first.
- **Family double-fire returns at new grain** (Henry vs Bachman/Reilly; brigade events over Long Lane skirmishers): validator can't read attestation — the per-wave review checklist item is the guard, same discipline Phase 6 Task 2 established.
- **Deferred honesty debt:** HQs/hospitals and the off-map annotation layer are deliberately absent; the survey holds their complete research base and this plan's decisions name the phase that owns each. The risk is forgetting — the memory pickup doc entry in Task 9 is the tether.
