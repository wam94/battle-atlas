## The two HDRP quality profiles and how to switch

Both live in `app/Assets/Settings/HDRP/` and are validated by the
`HdrpMigrationTests` EditMode suite:

| Profile | Asset | Shape |
| --- | --- | --- |
| **PLAYBACK** (project default) | `BattleAtlasHDRP_Playback.asset` | Real-time Atlas: SSR off, SSAO off, volumetrics off, R11G11B10 color, motion vectors on |
| **OFFLINE** | `BattleAtlasHDRP_Offline.asset` | §3.2 offline render: SSR + SSAO + volumetrics + motion vectors (Recorder-accumulation-ready), R16G16B16A16 color |

Volume pairing: the Atlas scene's Global Volume uses
`AtlasVolumeProfile.asset` (sky/exposure/post — see calibration below);
`OfflineRenderVolumeProfile.asset` layers volumetric fog (mean free path
8 km — dense enough to exercise §3.2 volumetrics, thin enough that the
4 km macro orbit view stays readable; Phases 7/9 author their own
eye-level volumes), SSAO/SSR overrides, and a longer shadow reach on top.

Switching:

- **Committed default:** playback (nothing to do).
- **Interactive inspection:** `BattleAtlas ▸ HDRP ▸ Use Offline Profile
  (Session)` / `Restore Playback (Session)` — session-only quality-level
  override, nothing saved.
- **Headless offline render:** `AtlasHdrpRender.CaptureDeterminismPair`
  binds the offline asset + offline volume itself; no settings flip.
- The migration itself is re-runnable:
  `BattleAtlas ▸ HDRP ▸ …` menu items or
  `-executeMethod BattleAtlas.EditorTools.HdrpMigration.MigrateAll`.

**Active build target must support HDRP.** The owner's editor was last on
iOS; HDRP refuses to construct there ("Platform iOS is not supported with
HDRP") and everything renders blank. Switch the editor to
**macOS (OSXUniversal)** — all Phase 4 CLI evidence runs pass
`-buildTarget OSXUniversal`. iPhone support is explicitly deferred by the
plan (§2), so this costs nothing now.

## Exposure calibration (the "HDRP reads dim/flat" fix)

The Gate P3 bake-off established HDRP physical light units + explicit
exposure as the calibration path. Phase 4 locks it in:

- **Sun:** physical directional light, **100,000 lux** base (clear July
  afternoon), angular diameter 0.53°. `SunDirector`'s elevation-keyed ramp
  (1.0 at ≥69.4° elevation → 0.88 at 37.9°) scales it every frame from the
  battle clock, so t=8400 (15:20 LMT) reproduces the approved bake-off's
  ~95k lux.
- **Exposure: FIXED 13.2 EV100** (sunny-afternoon), in
  `AtlasVolumeProfile`. Fixed, not automatic: automatic exposure is
  content-dependent and would make offline renders nondeterministic and
  playback/offline inconsistent.
- **Sky:** physically based sky, dynamic ambient.
- **Post:** the URP Atlas chain carried over unchanged (Neutral tonemap,
  bloom 0.25, vignette 0.2) so the before/after comparison isolates the
  pipeline, not a re-grade.

**Custom-shader lighting contract:** the BattleAtlas hand-written shaders
(`SoldierVertexTint`, `Flag`) do NOT participate in physical
lighting/exposure. They read the deterministic globals `_BattleSunDirWS` /
`_BattleSunColor` (normalized URP-scale intensity, published by
`SunDirector` from the battle clock) and write display-intent values into
HDRP's pre-exposed color buffer. That keeps the Atlas's authored figure
and flag legibility byte-stable under any volume EV, and keeps every
animated input on the battle clock (never `_Time`).

## Shader conversions

| Shader | Approach | Determinism | Instancing |
| --- | --- | --- | --- |
| `BattleAtlas/SoldierVertexTint` | Second SubShader (hand HLSL, `RenderPipeline=HDRenderPipeline`, one `ForwardOnly` pass); vertex-color bands × MPB `_BaseColor` unchanged; half-lambert (0.35 floor) against the sun globals | `_BattleSunDirWS`/`_BattleSunColor` from SunDirector (clock-driven); no `_Time` | `multi_compile_instancing` + `RenderMeshInstanced` + per-batch MPB verified |
| `BattleAtlas/Flag` | Second SubShader, same pattern, `Cull Off`; identical wave math | `_BattleWaveTime` global (BattleDirector, battle clock); per-flag phase now uses `GetObjectAbsolutePositionWS()` because HDRP is camera-relative — raw matrix translation would have tied the cloth phase to the camera | same |

The URP SubShaders remain in both files (Unity selects by pipeline tag),
so the preserved URP history still renders if a URP asset is bound.

Neither pass declares a ShadowCaster — identical to the URP originals (no
regression; figures gain shadow casting when Phase 8 revisits shadows).

## Terrain conversion

`Assets/Generated/LandcoverTerrain.mat` swapped to **HDRP/TerrainLit in
place** (GUID preserved, so the committed Atlas scene reference and the
gitignored generated terrain survive untouched); `LandcoverImporter` now
emits HDRP/TerrainLit for fresh imports. The two load-bearing URP tricks
carry over verbatim:

1. **Relief-in-albedo** is baked into the layer tint textures by the
   importer — shader-independent by construction; HDRP samples the same
   4-layer splat (well under HDRP's 8-layer/pass limit).
2. **alpha=0 anti-gloss**: HDRP TerrainLit, like URP, derives layer
   smoothness as `albedo.a * _Smoothness{i}` when a layer has no mask map
   (`TerrainLit_Splatmap.hlsl`, `DefaultMask`) — ReliefDecoder's alpha=0
   texels and the belt-and-suspenders `TerrainLayer.smoothness = 0` both
   still force matte ground.
3. Per-pixel instanced normals: property and keyword names are identical
   in HDRP (`_EnableInstancedPerPixelNormal`,
   `_TERRAIN_INSTANCED_PERPIXEL_NORMAL`); `drawInstanced` stays on.

Before/after: compare `docs/benchmarks/captures/atlas-t*.png` (P0, URP)
against `hdrp-atlas-t*.png` — relief shading, swale darkening, and matte
field response are the review points.

### Deterministic offline frame (Gate P10 standard)

`AtlasHdrpRender.CaptureDeterminismPair`: Atlas at t=8700, OFFLINE
profile, 2560×1440, 3 warmups then two submissions, raw RGBA32 compare:

- `hdrp-atlas-offline-t8700-a.png` / `-b.png`
- `hdrp-atlas-determinism.json`

Measured (Apple M4, Metal, 6000.4.11f1): **10,936 / 3,686,400 pixels
differ (0.297%), max channel delta 10/255**. Attribution: HDRP's
volumetric fog integrates with per-frame temporal jitter (and SSAO is
temporally accumulated), so consecutive submissions of the same logical
frame sample slightly different noise phases — with the offline volume's
volumetric fog OFF, the same pair measured 401 pixels (0.011%), max
delta 7. All scene logic (unit poses, casualty state, smoke matrices,
sun) is a pure function of the battle clock and reproduces exactly; the
residual is GPU/temporal-effect noise, invisible at 1:1, and is recorded
here as the Gate P10-standard tolerance. Phase 10's Recorder
accumulation path averages exactly this class of jitter (and can freeze
the volumetric frame index if bit-exactness is later required).
